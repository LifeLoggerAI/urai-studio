# URAI Foundation Integration

URAI Studio uses URAI Foundation as the review anchor for public-facing creative assets, accessibility, provenance, and responsible content claims.

## Canonical references

- Foundation repository: `LifeLoggerAI/urai-foundation`
- Public domain target: `https://uraifoundation.org/`
- Governance charter: `https://uraifoundation.org/docs/governance-charter.md`
- Ethical AI principles: `https://uraifoundation.org/docs/ethical-ai-principles.md`
- Transparency framework: `https://uraifoundation.org/docs/transparency-framework.md`
- Risk review process: `https://uraifoundation.org/docs/risk-review-process.md`
- System-of-systems contract: `https://uraifoundation.org/docs/system-of-systems-integration.md`

## Studio alignment requirements

Studio changes should reference Foundation standards when they affect:

- public launch assets;
- claims about human benefit, consent, privacy, or safety;
- accessibility of visual, motion, audio, or interactive materials;
- provenance, ownership, or licensing metadata;
- sensitive depictions of users, relationships, emotions, homes, workplaces, or vulnerable communities.

## Release gate

For major launch assets, complete an accessibility review and check public claims against Foundation standards before release.

## Live-domain caveat

Do not treat `uraifoundation.org` as live on GitHub Pages until DNS no longer resolves to Squarespace and `/sitemap.xml` returns `200`.
